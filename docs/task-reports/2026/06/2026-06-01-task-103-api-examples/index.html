Grimoire Task Report

Human Readable API Examples

2026-06-01 TASK-103 Completed

Before

API.md documented schemas and some curl snippets, but examples were request-only and many local-client workflows had no generated response body or error-flow evidence.

Problem

Grimoire parity calls for human-readable API examples generated from the contract, with enough request and response context for local scripts and clients to integrate without hand-maintained prose.

Changes Added

  • Extended API examples with optional response status, content type, headers, and JSON/text body fields.
  • Updated the Markdown generator to render Request and Response blocks from the source contract.
  • Added curated examples for integration tokens, MCP auth failure, bookmark create/list/detail/update, search filtering and pagination, import/export, categories, tags, backup, and common 4xx flows.
  • Regenerated API.md and docs/api-contract.json so drift checks cover the examples.

Result

Local client authors now see concrete curl commands next to representative HTTP responses, including bearer-token challenges, validation failures, paginated list envelopes, and generated contract JSON examples.

Verification

  • npx vitest run scripts/api-docs-generator.test.ts failed RED with missing response examples, then passed with 8/8 tests after implementation and review fixes.
  • npm run docs:api regenerated API.md and docs/api-contract.json.
  • npm run check passed, including lint, frontend and daemon type-checks, frontend tests, daemon tests, API docs drift check, and production build. Existing Fast Refresh, React Router future-flag, Browserslist, and chunk-size warnings remain.
Implementation Notes

Examples stay attached to the source contract instead of the generated Markdown. The generated docs intentionally do not invent sort query examples because the current contract has no server-driven sort parameter; TASK-115 owns that API surface.
