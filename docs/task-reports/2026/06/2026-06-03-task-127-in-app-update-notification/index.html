In-App Update Notification

P1 TASK-127 done

Added a non-intrusive in-app notification banner that alerts users when a newer version of Little Imp is available. The banner appears on the main library page, is dismissible, stays dismissed per version, and links to the Settings → Updates section for detailed update information.

What was implemented

src/components/UpdateAvailableBanner.tsx

A new banner component rendered with an amber/attention color scheme, showing "Little Imp {tag} is available (current: {version}) — View update". Includes a dismiss button. Clicking "View update" navigates to /settings using React Router.

src/hooks/use-update-check.ts

A background update-check hook that:

  • Calls GET /updates/check once on app load
  • Debounces subsequent checks to a 6-hour interval (stored in localStorage)
  • Silently ignores daemon-unreachable errors (no banner shown)
  • Tracks dismissed version in localStorage so banner stays hidden per version
  • Never auto-downloads or auto-installs

src/pages/Index.tsx

Wired the update check hook and banner into the main library page. The banner renders after the existing DaemonOfflineBanner and DegradedModeBanner, inheriting the same layout and spacing.

e2e/update-notification.spec.ts

Playwright e2e tests covering:

  • Banner visibility when update is available
  • No banner when up to date
  • No banner when daemon is unreachable
  • Dismiss and persist across page reload
  • "View update" navigation to Settings page

src/components/UpdateAvailableBanner.test.tsx

Component unit tests for version rendering, dismiss callback, and accessibility role.

src/pages/Index.test.tsx

Integration tests verifying the banner appears/disappears based on hook state, and that dismiss calls propagate correctly.

Design decisions

  • Plain fetch instead of apiFetch helper: The existing checkForUpdates() returns a deeply-nested generic type (ApiDto) that triggers TypeScript "excessively deep" errors. Using fetch directly with a locally-defined UpdateCheckResult interface avoids the generic instantiation entirely while keeping the hook decoupled from the daemon type system.
  • 6-hour debounce via localStorage: Prevents unnecessary network requests on every page load while surviving page refreshes within the same session. The debounce timestamp is stored in localStorage at littleimp_update_last_check_ms.
  • Version-aware dismissal: Dismissing the banner stores the dismissed release tag (e.g. v0.2.0) in localStorage, so the banner stays hidden for that version but re-appears for a newer one.
  • Amber color scheme: A soft amber/attention palette (matching the existing Settings update-check result card) signals "action recommended" rather than "error" or "success", which keeps the banner informative without being alarming.
